# 写作规范

这份规范用于让 Wiki 内容更容易检索、维护和协作。

## 标题与结构

- 每个页面只使用一个一级标题（`# 标题`）
- 用二/三级标题（`##` / `###`）组织小节
- 段落尽量短：一句话一个观点

## 命名建议

- 文件名用小写 + 连字符：`my-topic.md`
- 目录按主题划分：`guide/`、`reference/`、`howto/` 等

## 常用写法

### 提示块

::: tip 小技巧
这里写提示内容。
:::

### 代码块

````markdown
```powershell
npm run docs:dev
```
````

## 链接

- 站内链接优先用相对路径：`../reference/faq.md`
- 外部链接建议写清楚标题，避免只贴 URL
