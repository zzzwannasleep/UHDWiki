# 常见的报错代码与原因

本页整理常见报错码的含义与排查思路。不同客户端的提示可能不同，但大多数可以归类为 **账号/权限**、**链接/资源**、**网络**、**服务端异常** 或 **解码兼容**。

## 常见 HTTP 状态码

| 代码 | 常见含义 | 你可以先做什么 |
| --- | --- | --- |
| 400 | 请求参数不正确 | 刷新页面/重试；换客户端验证 |
| 401 | 未登录/登录过期 | 重新登录；检查账号是否被踢下线 |
| 403 | 权限不足/访问被拒 | 确认套餐/权限；确认是否触发风控/限制 |
| 404 | 资源不存在/链接失效 | 资源可能下架或链接已变；返回上一页重试 |
| 429 | 请求过多/频率限制 | 等待一段时间；减少并发/频繁刷新 |
| 500 | 服务端内部错误 | 稍后重试；换资源/换线路（如有） |
| 502/503 | 网关/服务不可用 | 服务端波动；稍后再试 |
| 504 | 网关超时 | 网络不稳定或服务端响应慢；稍后再试 |

## 播放器常见报错类型

- **Network Error / 连接失败**：网络抖动、DNS、代理/VPN、链路质量问题
- **Decode Error / 解码失败**：设备不支持该编码/规格（例如 HEVC 10bit/HDR）
- **DRM / Not supported**：当前环境不支持所需的播放能力（如某些浏览器限制）

## 快速排查（推荐顺序）

1. **刷新/重试**：排除偶发网络抖动
2. **重新登录**：尤其是 401/403 相关提示
3. **换一个客户端或设备**：快速判断是兼容性还是账号/服务端问题
4. **换一个清晰度/版本**：判断是否为解码/带宽问题
5. **稍后再试**：500/502/503/504 很可能是服务端波动

## 反馈时建议提供的信息

- 报错码/报错文案截图
- 片名 + 集数/时间点
- 使用的客户端（网页/手机/电视/PC）与系统版本
- 网络环境（Wi‑Fi/有线/5G、是否使用代理/VPN）
