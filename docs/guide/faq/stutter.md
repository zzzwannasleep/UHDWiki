# 常见的卡顿原因

卡顿/缓冲通常不是单一原因。先确认是 **网络问题**、**服务端问题** 还是 **设备解码问题**，再对症处理。

## 最常见的原因

- **带宽不够**：4K/HDR 高码率资源对网络要求很高
- **网络抖动/丢包**：测速够但播放仍频繁缓冲
- **服务端转码压力大**：需要转码时，服务器性能不足会导致卡顿
- **设备硬解不支持**：部分设备对 HEVC/10bit/HDR 支持不完整
- **字幕导致转码/烧录**：某些字幕格式或配置会触发转码
- **后台占用**：下载/同步/多设备并发占用带宽或并发数

## 快速排查（推荐顺序）

1. **先换一个清晰度/版本**：从 4K 切到 1080P 试一下，快速判断是否为带宽/解码瓶颈
2. **测速并观察稳定性**：除了下行速度，也关注延迟与丢包（Wi‑Fi 信号弱也会卡）
3. **换网络环境**：Wi‑Fi ↔ 有线 ↔ 5G，排除路由器/运营商链路问题
4. **确认是否在转码**：如果能看到 “Direct Play/Direct Stream/Transcode” 之类信息，优先争取 Direct Play
5. **降低高成本功能**：如强制高码率、强制 HDR、实时字幕烧录等
6. **更新播放器/系统**：旧版本可能存在解码或 HLS/DASH 兼容问题

## 常见解决思路

- **优先使用支持硬解的客户端/设备**（新系统/新设备通常更稳）
- **选择更适配设备的版本**（例如 SDR / 1080P）
- **尽量避免弱 Wi‑Fi**：距离远、隔墙多时，卡顿概率明显上升

## 反馈时建议提供的信息

- 片名 + 集数/时间点
- 播放端（网页/手机/电视/PC）与系统版本
- 清晰度与规格（1080P/4K、HDR/SDR 等）
- 是否出现报错码/截图
- 网络环境（Wi‑Fi/有线/5G）与大致测速结果
